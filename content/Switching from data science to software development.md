# Switching from data science to software development

This is the core note on my shift away from data science and toward actual software engineering work.

## Notes

I was a final-year B.Tech Computer Science student in Hyderabad, India, actively chasing a software development role paying 6+ LPA. I shared my resume, which was still leaning toward data science, and made it clear that I wanted to move hard into software engineering because I find data science boring and actually enjoy building software. covered career strategy, language selection for DSA practice, resume optimization, and technical DSA concepts. The main career decisions reached were: learn DSA using C++ (chosen for personal enjoyment over Java, despite Java having more job openings in India), then pick up Java and Spring Boot in week three of a one-month plan to maximize employability. The plan involves building a Spring Boot Task Management REST API project to complement existing projects ([[SelfForge]], an AI-powered productivity platform, and an Interactive Knowledge Graph Portfolio). Claude also created a fully rewritten SDE-focused resume as a DOCX file, removing all data science positioning and reframing the internship title from "Data Analyst" to "Software Development Intern." Technical topics covered included linked list traversal patterns in both C++ and [[Python]], the LeetCode problem "Add Two Numbers," the dummy node and tail pointer pattern for O(n) list construction, and carry propagation logic. I preferred a simple-first explanation before optimization. I also received curated English-language learning resources (The Cherno for C++, NeetCode, William Fiset, Amigoscode for Spring Boot) with a preference for non-Indian content creators. Toward the end I got an automated values assessment email from HSBC for a Data Analyst role, and the obvious move was to ignore it and keep focusing on SDE roles.
