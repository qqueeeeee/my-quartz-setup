# Keyboard Centric Workflow

Everything on my computer is based on this idea.

I hate having to constantly move my hand back to the mouse just to do basic stuff. If I can keep most interactions on the keyboard, everything feels faster, cleaner, and way less annoying.

## What that actually means for me

This usually shows up through a bunch of tools and habits:

- [[Tiling Window Managers]] because I like my screen layout being something I can control instantly
- [[GlazeWM]] because I still want that kind of workflow on Windows
- [[Neovim]] because it feels way more natural to me than clicking around an IDE
- [[Quartz]] because even this site is built around shortcuts and keyboard-first navigation

I tried [Zed](https://zed.dev/) for a bit since it has a nice GUI and supports vim motions, but I still kept feeling like I was using a tool that looked right instead of a tool that actually fit me. Neovim just feels more native to how I already think.

## Why I care about this so much

It is not really about speed in the "save 0.4 seconds" kind of way.

It is more that keyboard-heavy workflows make the whole system feel more direct. Less friction, less context switching, less tiny interruptions.

That is also why this shows up outside my local setup too:

- in how I use [[How I use Obsidian|Obsidian]]
- in how I think about interfaces in [[Quartz]]
- in the kind of tools I want to build for myself, like [[SelfForge]]
