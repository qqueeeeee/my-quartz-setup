# Vault Style Guide

This is the basic style I want this vault to follow.

It is not some strict religion or anything, it is just the set of rules that makes the whole thing easier to maintain without thinking too much.

## Main idea

The vault should stay file-first, link-heavy, and as low-friction as possible.

I do not want to spend more time organizing notes than actually writing them.

## Rules I want to follow

- keep one vault
- keep folders to a minimum
- keep most notes in the root
- use internal links a lot
- use `YYYY-MM-DD` dates
- pluralize categories and tags when it makes sense
- prefer plain markdown and simple frontmatter

## Folder philosophy

Most notes in this vault should live in the root because they are either:

- things I wrote
- things I am learning
- project notes
- conversations / ideas / essays that relate directly to me

The folders that should exist are mostly just support folders:

- `References` for things outside my world like books, people, places, games, tools, etc.
- `Clippings` for things other people wrote
- `Templates` for templates
- `Daily` for dated daily-link notes if I use them
- `Attachments` for media and files

`Categories` can stay around as an overview layer, but it should not become the main way I think about the vault.

## Why this matters

I want structure to emerge from links, categories, and repeated themes, not from me endlessly deciding where a file belongs.

That is the whole point.
