# C++ Intro

I have always thought C++ was a cool language.

The first time I touched it was when I was trying to reverse engineer [[Superfighters Deluxe]]. That attempt failed pretty badly and I switched to C# later, but that short time with C++ still stuck with me because it showed me how much control the language gives you.

Stuff like DLL injection and messing with running programs felt insane to me at the time. It made programming feel way less abstract and way more real.

## Why I want to learn it properly

The main reason is simple: I want a better understanding of what is actually happening under the hood.

C++ forces you to care about things like:

- memory
- compilation
- references and pointers
- stack vs heap
- performance as an actual design constraint

That is also why this connects pretty naturally to [[Writing a ML Library in C]]. If I am going to build lower-level systems, I do not want to only understand them from the outside.
