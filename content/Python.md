# Python

I mostly like Python when I want to build something quickly and get straight to the logic.

It is not the language I think is the coolest, but it is one of the easiest to actually move fast in, and that matters a lot for projects where I care more about iteration than low-level control.

That is part of why it makes sense for [[SelfForge]] too. I want to spend time on the AI/system side of the product, not fight the language every five minutes.

## Where it shows up for me

- scripting
- AI tooling
- backend / app logic
- quick experiments

It also pairs nicely with lower-level learning because it gives me a contrast:

- [[Python]] when I want speed of building
- [[C++]] or [[Writing a ML Library in C]] when I want deeper control or understanding
