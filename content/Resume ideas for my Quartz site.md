# Resume ideas for my Quartz site

This is about how a resume should live inside [[Quartz]] without feeling bolted on.

## Notes

If I add a resume to [[Quartz]], it should not feel like some boring external attachment sitting next to the real site.

The best version is making the resume part of the graph itself.

That could mean:

- a dedicated resume note
- a downloadable PDF alongside it
- resume sections linking directly into project and skill notes
- a graph node that acts like the structured entry point for recruiters

That fits the site way better than just dropping a random navbar button that leads to a dead-end page.

This also connects to a bigger note-taking problem: if I want the site to stay alive, I need to keep feeding it notes that actually come from my work.

The easiest source material is usually:

- project decisions
- bugs I fixed
- things I learned while building
- ideas I ran into and want to come back to later
